## ‚ö° What I‚Äôm about

I build systems where **trust is a feature**, not a marketing line:
- **Cryptographic proof** over vibes
- **Auditable by default** (logs that actually help during incidents)
- **Deterministic behavior** under load (money math included)
- **Security boundaries** made explicit (and tested)

If it needs to be fast, safe, and boring to operate, that‚Äôs my zone.
